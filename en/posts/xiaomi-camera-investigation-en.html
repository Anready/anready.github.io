Introduction

Modern smartphones increasingly offer advanced camera systems, but their functionality can be limited not only by hardware but also by software. In this article, we investigate how Xiaomi restricts camera capabilities in the Redmi 13, which uses the same Samsung HM6 108MP sensor as the Redmi Note 13, yet lacks a PRO mode. Through analysis of the Camera2 API and experiments, we demonstrate how software limitations impact functionality and image quality, and propose potential solutions for users.

The Problem: Absence of PRO Mode in Redmi 13

The Redmi 13 and Redmi Note 13 are equipped with the same Samsung HM6 108MP camera sensor, which supports high-quality imaging and manual settings such as ISO, shutter speed, and white balance. However, the default camera app on the Redmi 13 lacks a PRO mode, which enables these manual controls, while the Redmi Note 13 includes it. This raises the question: are these limitations due to hardware or software?

For clarity, here are the key specifications:

Model Sensor PRO Mode Max Shutter Speed (Official)
Redmi 13 Samsung HM6 No 1 s
Redmi Note 13 Samsung HM6 Yes 30 s

References: Xiaomi Redmi 13 review, Xiaomi Redmi Note 13 4G review

Verifying Camera2 API Support

To determine whether the Redmi 13’s camera supports PRO mode at the hardware level, we used the Camera2 Probe app, which analyzes the level of Camera2 API support. The results confirm that the Redmi 13’s camera supports the full Camera2 API (Level 3), including manual controls for ISO, shutter speed, focus, and white balance — everything required for a PRO mode.

This indicates that the absence of PRO mode in the Redmi 13’s default camera app is a software restriction, not a hardware limitation.

Experiments with Third-Party Apps

To test our hypothesis, we installed a third-party camera app with PRO mode support (e.g., Open Camera) on the Redmi 13. The app successfully enabled manual controls, including ISO and shutter speed, confirming hardware compatibility. However, capturing images at the full 108MP resolution remained unavailable through third-party apps, likely due to API restrictions at the driver level.

We also tested the maximum shutter speed via the Camera2 API. While the API officially reports a maximum of 1 second for the Redmi 13, bypassing this restriction programmatically allowed us to achieve a shutter speed of up to 30 seconds, matching the Redmi Note 13’s capabilities.

These experiments confirm that Xiaomi imposes software limitations on the Redmi 13’s camera, likely to differentiate it from higher-priced models like the Redmi Note 13.

Possible Reasons and Implications

Software restrictions may be part of Xiaomi’s strategy to promote more expensive models, such as the Redmi Note 13, where PRO mode and other features are fully accessible. This allows the company to offer similar hardware across different price segments while varying functionality.

For users, this means the Redmi 13’s camera potential is underutilized. The lack of PRO mode limits creative control in challenging lighting conditions, and restricted access to 108MP resolution in third-party apps reduces flexibility.

Solutions for Users

  1. Using Third-Party Apps: Apps like Open Camera or GCam can bypass some restrictions and enable PRO mode. However, access to the full 108MP resolution may still be limited.
  2. Custom ROMs: Installing custom firmware, such as LineageOS, may unlock additional camera features but requires technical expertise and voids the warranty.
  3. Developing a Custom App: For developers familiar with Java or Kotlin, creating a custom app using the Camera2 API can overcome limitations like the 1-second shutter speed cap. If this issue persists, refer to solutions 1 and 2.

Conclusion

Our investigation reveals that the Redmi 13’s Samsung HM6 camera sensor is fully capable of supporting PRO mode and other advanced features, but Xiaomi restricts these capabilities through software. This is evidenced by Camera2 API analysis and experiments with third-party apps. Such restrictions likely serve to differentiate models and promote higher-end devices.
